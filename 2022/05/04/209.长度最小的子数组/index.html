5.长度最小的子数组【滑动窗口】

209、给定一个含有 n 个正整数的数组和一个正整数 s ，找出该数组中满足其和 ≥ s 的长度最小的 连续 子数组，并返回其长度。如果不存在符合条件的子数组，返回 0。

示例：

输入：s = 7, nums = [2,3,1,2,4,3] 输出：2 解释：子数组 [4,3] 是该条件下的长度最小的子数组。

草稿：

  • 一个元素一个元素的查：
    • i循环每个是否有=s
  • 两个元素……：
    • i和i+1做和，i++
  • 三个……：
    • i，i+1，i+2，i++
  • 任何

滑动窗口

不断的调节子序列的==起始位置==和==终止位置==，从而得出我们要想的结果

滑动窗口也可以理解为==双指针法==的一种

实现滑动窗口，主要确定如下三点：

  • 窗口内是什么？
    • 满足其和 ≥ s 的长度最小的 连续 子数组
  • 如何移动窗口的起始位置？
    • 如果当前窗口的值大于s了，窗口就要向前移动了（也就是该缩小了）
  • 如何移动窗口的结束位置？
    • 窗口的结束位置就是遍历数组的指针

​ 思路：

  • 设置结果result为一个很大的数：INT32_MAX，因为要找最小的result
  • 滑动窗口三要素
    • 起始位置 0
    • 长度 sublength 0
    • 窗口中数值之和sum 0
  • 循环遍历，即j指针，从0到size()-1
    • 把j指针遇到的元素都加给sum
    • ==判断==，当sum>=s时（因为后面加的数可能很大，此时要减去前面的多个数，所以用while（）判断！）
    • 记录下当前窗口大小 j-i+1
    • 和每次的result比较，是否更新最小窗口result
    • 缩小窗口：
      • sum和-窗口起始指针位置
      • 起始指针后移i++
    • 👆先记录下当前，再缩小，否则缩小之后不知道之前哪个是多少了！
  • return
    • 如果result还是max，说明没找到一个sublength赋给result，返回0
    • 否则返回result

注意==是否相等，=赋值

lr指针，

==l不断扩大为了使sum>target==

==r不断缩小，为了找到最小！==

​ 所以l一找到一个合适的sum，就会判断此时的窗口大小是否最小，再让r也缩小，再判断能否满足>target

​ while中，主体是找sublength，后更新条件（r右移写在后面）

for+while：

  • O(n^2)
  • O(n)
    • 每一个元素被操作的次数，每个元素在滑动窗后进来操作一次，出去操作一次，每个元素都是被操作两次，所以时间复杂度是 2 × n 也就是 O(n)

904、水果成篮

你正在探访一家农场，农场从左到右种植了一排果树。这些树用一个整数数组 fruits 表示，其中 fruits[i] 是第 i 棵树上的水果 种类 。

你想要尽可能多地收集水果。然而，农场的主人设定了一些严格的规矩，你必须按照要求采摘水果：

你只有 两个 篮子，并且每个篮子只能装 单一类型 的水果。每个篮子能够装的水果总量没有限制。
你可以选择任意一棵树开始采摘，你必须从 每棵 树（包括开始采摘的树）上 恰好摘一个水果 。采摘的水果应当符合篮子中的水果类型。每采摘一次，你将会向右移动到下一棵树，并继续采摘。
一旦你走到某棵树前，但水果不符合篮子的水果类型，那么就必须停止采摘。
给你一个整数数组 fruits ，返回你可以收集的水果的 最大 数目。
