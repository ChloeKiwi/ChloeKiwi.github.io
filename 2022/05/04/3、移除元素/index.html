27. 移除元素

给你一个数组 nums 和一个值 val，你需要 原地 移除所有数值等于 val 的元素，并返回移除后数组的新长度。

不要使用额外的数组空间，你必须仅使用 $O(1)$ 额外空间并原地修改输入数组。

元素的顺序可以改变。你不需要考虑数组中超出新长度后面的元素。

示例 1: 给定 nums = [3,2,2,3], val = 3, 函数应该返回新的长度 2, 并且 nums 中的前两个元素均为 2。 你不需要考虑数组中超出新长度后面的元素。

示例 2: 给定 nums = [0,1,2,2,3,0,4,2], val = 2, 函数应该返回新的长度 5, 并且 nums 中的前五个元素为 0, 1, 3, 0, 4。

为什么返回数值是整数，但输出的答案是数组呢?

1 输入数组是以「引用」方式传递的

2 意味着在函数里修改输入数组对于调用者是可见的

数组的元素在内存地址中是==连续==的，不能单独删除数组中的某个元素，==只能覆盖==

(也就是如果在数组中某个位置找到元素，不能直接删除，“前赴后继”)

暴力解法

两层for循环，一个for循环遍历数组元素，第二个for循环更新数组

思路

  • 外层循环从0开始遍历，0，1，2…
  • 找到一个要删除的目标i后，将后面（从i+1开始）的元素集体向前移
    • 前移：nums[j-1]=nums[j]
  • 内层循环走完，要去外层循环了，i++
  • 但是还要检测当前i位置，所以i–
  • 经历了内层循环（移动位置）后size-1

外循环：遍历，找val

内循环：挨个移动后面元素

时间复杂度：o（n^2），两层循环，都是n数量级

空间复杂度：o（1），没有会变化的变量

双指针法（快慢指针法）

通过一个==快指针==和==慢指针==在==一个for==循环下完成两个for循环的工作

应用场景：数组、链表、字符串

思路

明确：val/target表示要去掉的、不要的

  • 循环for控制快指针，+1，+1
  • 快指针找是否是val
    • 是：
    • 快指针+1，直接跳过（相当于去掉这个元素），慢指针不变/不动
    • 否：
    • 快指针的值赋给慢指针（慢指针数组+1），慢指针+1，快指针+1
  • return slow：slow数组里正好都是需要的，不需要的fast都直接略过了

写错了：

赋值是一个等号！

时间复杂度：n

空间：1

不要使用额外的数组空间，你必须在 原地 修改输入数组 并在使用 O(1) 额外空间的条件下完成。

=：跳过，

！=：赋给慢指针

错误1：

要考虑这种情况，因为我们的初始从1直接开始了

==巧思==：

不用再管0位置是什么，因为0一定在slow中

时间：n，空间：1

977、有序数组的平方

给你一个按 非递减顺序 排序的整数数组 nums，返回 每个数字的平方 组成的新数组，要求也按 非递减顺序 排序。

平方–>从小到大排序

思路：

  • 先计算每个数的平方
  • 存在新数组里nums_2
  • 遍历新数组，第一个为max
    • 后面的每一个比xxxxxx

暴力排序

  • 遍历每个nums[]元素

  • 平方 A[i]*=A[i]

  • sort(A.begin(),A.end());//快排

时间：n+nlog2n，空间：1

sort()基本使用方法

  • 在C++中使用sort()函数需要使用#include<algorithm>头文件

  • sort()函数可以对给定区间所有元素进行排序。它有三个参数sort(begin, end, cmp)

    • end为指向待sort()的数组的最后一个元素的下一个位置的指针比如，==nums.end()取最后一个元素==

    • cmp参数可以不写，默认从小到大（排序规则）

    • 将cmp参数写为==greater()==就是对int数组进行从大到小排序

      • <>中我们也可以写==double、long、float==等

      • ```cpp
        int num[10] = {6,5,9,1,2,8,7,3,4,0}; sort(num,num+10,greater());
